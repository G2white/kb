# 供应链速查库 板块设计文档

- 日期：2026-05-29
- 状态：已定稿（原型已实现）
- 所属项目：线上知识库 `kb-site`（GitHub Pages: https://g2white.github.io/kb/）

## 1. 目标与背景

在现有知识库中新增「供应链速查库」板块，把展会上收集的供应商资料结构化整理。

**核心用途**：下次做产品时，能按「能做什么 / 做什么产品 / 用在什么场景」快速定位到合适的供应商，并立刻看到其能力、产品性能、商务信息与联系方式。

**关键约束**：
- 供应商数量会**持续增长**，因此筛选/搜索能力必须从一开始就内建，加公司不需返工。
- 复用现有报告页的**深色主题**设计语言。
- 纯静态站点，**GitHub Pages 与本地双击（file://）都要能正常工作**。

## 2. 架构决策

采用**方案 A：数据驱动首页 + 每公司独立详情页**。

- 一份结构化数据 `companies.js` 作为**索引单一数据源**，驱动首页的搜索与筛选。
- 每家公司一个**富内容详情页** HTML，承载照片、PDF、名片等多媒体资料。
- 轻微取舍：公司名/标签会在 `companies.js` 与详情页各写一遍（可接受），换取干净的筛选实现与可靠的程序化录入。

被否决的方案：
- 方案 B（所有公司塞进单个大文件）：增长后臃肿、富媒体难管理。
- 方案 C（手写卡片 + data 属性筛选）：手工维护易错、规模大后首页冗长。

## 3. 目录结构

```
kb-site/
  index.html                      # 知识库首页（含「供应链速查库」入口卡片）
  supply-chain/
    index.html                    # 供应商目录页：搜索 + 三类标签筛选 + 卡片网格
    companies.js                  # 全部公司索引数据（单一数据源，驱动筛选）
    style.css                     # 共享深色主题（首页 + 详情页共用）
    companies/
      <slug>.html                 # 每个公司一个详情页
    assets/
      <slug>/                     # 该公司专属资料：照片 / PDF / 名片扫描件
```

## 4. 命名与技术约定

- **slug 三位一体**：`companies.js` 的 `slug` = 详情页文件名 `<slug>.html` = 资料文件夹 `assets/<slug>/`。三者必须严格对应。
- slug 用**英文小写 + 连字符**（如 `acme-motors`），避免中文路径在 GitHub Pages 上的编码问题；页面显示文字照常用中文。
- `companies.js` 通过 `<script src>` 加载（**不用 fetch JSON**），保证本地 `file://` 双击也能筛选，不被浏览器跨域限制阻挡。
- 样式集中在 `style.css`，所有页面 `<link>` 引用同一份，改样式只改一处。
- 空的 `assets/<slug>/` 子文件夹用 `.gitkeep` 占位以便 git 跟踪；放入真实文件后该占位可留可删。

## 5. 数据模型（`companies.js`）

```js
window.COMPANIES = [
  {
    slug: "acme-motors",                  // 唯一ID = 详情页文件名 = assets 子文件夹名
    name: "Acme Motors",                  // 公司名
    nameZh: "艾克米动力",                  // 中文名（可选）
    region: "中国 · 浙江",                 // 地区（仅展示，不参与筛选）
    summary: "一句话定位（卡片上显示）",
    capabilities: ["电机", "电控"],        // 维度①：能力/工艺（筛选）
    categories:   ["传动系统"],            // 维度②：产品类别（筛选）
    scenarios:    ["农业", "娱乐越野"],     // 维度③：应用场景（筛选）
    detail: "companies/acme-motors.html",  // 详情页相对链接
    source: "2025 广交会",                 // 来源展会（可选）
    addedDate: "2026-05-20"               // 收录日期
  }
];
```

- 首页**自动**从所有公司的三个标签数组汇总出筛选选项，无需单独维护标签清单。
- 搜索匹配范围：公司名 / 中文名 / 简介 / 地区 / 全部标签。
- 标签筛选语义：**同维度内多选做交集**（如同时选「农业」「娱乐越野」= 两者都满足）；跨维度也为交集。

## 6. 首页（目录页）交互

- 顶部面包屑返回知识库首页。
- 搜索框：实时过滤。
- 三个筛选分组（能力/工艺、产品类别、应用场景），每个是一排可点击 chip，选中高亮、再点取消。
- 结果计数 + 「清除筛选」按钮。
- 卡片网格：每张卡片显示公司名、中文名/地区、一句话简介、三色标签（蓝=能力 / 紫=产品 / 绿=场景），点击进详情页。
- 无匹配时显示空状态提示。

## 7. 详情页模板版块（顺序）

1. **顶部**：面包屑 + 公司名（中文名）+ 一句话定位 + 三色标签 + 地区/来源/收录日期。
2. **速览卡片**（hero-grid）：主营、起订量 MOQ、打样/交期、认证等关键指标。
3. **能力 / 工艺**：要点列表。
4. **主要产品 & 性能**：表格（产品 / 关键参数 / 适配场景 / 备注）。
5. **应用场景**：文字描述。
6. **联系方式** + **商务信息**（两栏）：联系人/电话/邮箱/网站/微信/WhatsApp；报价区间/起订量/交期/付款。
7. **资料附件**：产品照片画廊（`assets/<slug>/`）+ 文件/名片列表（PDF、名片图）。
8. **我的评估 / 备注**：callout（倾向合作 / 待评估 / 备选），记录主观判断与风险点。

> 商务信息（报价/MOQ/交期）放在详情页，不进首页筛选维度——筛选只用能力/产品/场景三维。

## 8. 录入一家公司的工作流（标准操作）

1. 取一个英文 slug（如 `xyz-electronics`）。
2. 在 `companies.js` 顶部追加一个公司对象，填全字段（尤其三个标签数组）。
3. 复制任一 `companies/<slug>.html` 作为模板，改名为新 slug，替换内容。
4. 建 `assets/<slug>/` 文件夹，放入照片、PDF、名片扫描件。
5. 在详情页把占位框换成真实 `<img src="../assets/<slug>/xxx.jpg">`，把文件链接指向真实文件。
6. 提交并推送，GitHub Pages 自动更新。

## 9. 示例数据的处理

原型内置 3 家示例公司（Acme Motors / IronFrame Chassis / Terra Tire）及首页/详情页的「示例占位」提示条，仅用于预览。**录入真实公司后**：删除这些示例对象与对应详情页、移除「示例占位」提示条（`.demo-note`）。

## 10. 不做的事（YAGNI）

- 不做按地区筛选（用户明确不需要）。
- 不做后端/数据库/构建步骤，保持纯静态手工/AI 维护。
- 不做账号权限——这是个人知识库，公开只读即可。
